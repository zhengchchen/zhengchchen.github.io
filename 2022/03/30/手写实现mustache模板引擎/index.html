手写实现mustache模板引擎

发表于 2022-03-30 | 分类于 Vue ， Vue源码 |
字数统计: 2.8k | 阅读时长 ≈ 11

前言

模板引擎是将数据要变为视图最优雅的解决方案，历史上出现过多种多样的模板技术，其本质都是将模文件和数据通过模板引擎生成最终的HTML代码。mustach是JavaScript中一种没有逻辑的模板语法，它可以应用在HTML、配置文化和源代码等任意地方。它通过使用散列或对象中提供的值在模板中展开标记来工作。

历史上的模板技术

  • 纯DOM法： 非常笨拙，没有实战价值
  • 数组join法： 曾几何时非常流行，是曾经的前端必会知识
  • ES6的反引号法： ES6中新增的${a}语法糖，很好用
  • 模板引擎： 解决数据变为视图的最优雅的方法

mustache使用

mustache是”胡子”的意思，因为它的嵌入标记{{}}`非常像胡子。`{{}}的语法也被vue所沿用。

  • templateStr : 模板字符串；
  • Data：需要嵌入模板字符串的数据。

mustache实现原理

什么是tokens？

  • tokens是一个JavaScript的嵌套数组，说白了，就是模板字符串的JavaScript表示；
  • 它是”抽象语法树“、”虚拟节点“等等的开山鼻祖，相当于编译原理中，解析的词法分析结果；
  • tokens也会产生嵌套的情况。

手写实现mustache库

开发环境

  • 模板化打包工具webpack(webpack-dev-server)、rollyp、Parcel等

  • mustache官方库使用rollup进行模块化打包，而本文采用webpack进行模块化打包，这是因为webpack能让我们更方便地在浏览器中（而不是nodejs环境中）实时调试

  • 生成库是UMD的，既可以在nodejs环境中使用，也可以在浏览器环境中使用

主要函数介绍

  • index.js ：主入口（包含测试案例）

  • lookup.js ：寻找对象中指定的值

  • nestedTokens.js： 将一维 tokens 折叠为多维 tokens

  • parseTemplateTokens.js ：将 template 字符串转换为 token

  • parseTokens.js ：解析循环的 tokens

  • renderTemplate.js： 获得最终的 template 和 data 结合的 DOM 字符串

  • Scanner.js ：Scanner 类对模板字符串进行扫描

实现Scanner类

Scanner类即一个扫描器，用来扫描模板字符串，有两个属性和三个方法：

  • pos：记录字符串当前正被扫描的位置

  • tail：记录当前指针及当前指针之后的位置的所有字符串

  • scanUntil(stopTag)：扫描模板字符串，直到遇见第一个stopTag结束，返回扫描过的字符串，此时调用scan()跳过stopTag

  • scan()：让指针跳过stopTag

  • eos()：判断当前字符串是否扫描完毕

实现ParseTemplateToTokens(templeteStr)函数

ParseTemplateToTokens的作用是将模板字符串变为tokens，过程中用到了上述的Scanner类和nestToken类（后文介绍），有三个属性：

  • tokens：存储tokens结果

  • scanner：用于扫描字符串，跳过结束关键字

  • words：用来保存scaner类扫描过的文字，在合适的地方将文字存入tokens数组

实现nestTokens()函数

  • 如上节所述，在遇到#字符时，需要进行嵌套存储。nestTokens类用于将一个未嵌套过的tokens处理成嵌套后的nestedTokens返回。

  • 这里，我们需要将#和/之间的字符串放入到当前token数组的第三项，作为它的嵌套项(tokens数组包含若干个token对象，每个token对象又包含类型和内容两个属性，如果当前token又嵌套内容，则单独开辟第三项用于存储)

如何实现嵌套循环？

引入栈和收集器(collector)，一开始，让collector指向nestedTokens结果数组，它是一个引用类型值，遇见#时，将当前的token 放入栈中，并给token添加下标为2的项，并且让collector指向它：collector = token[2] = [],遇见/时，先将栈顶元素退栈，此时改变collector为栈顶元素下标为2的数组，如果栈顶此时无元素，则证明当前的嵌套循环以及完成闭合或者无嵌套，可以让collector重新指向结果数组 collector = sections.length > 0 ? sections[sections.length - 1][2] : nestedTokens;

实现renderTemplate(tokens, data)函数

  • 此时我们已经利用ParseTemplateToTokens(templeteStr)函数得到了tokens数组，现在需要结合数据data将tokens对应部分填充，最终得到dom字符串。

  • renderTemplate(tokens, data)函数主要通过判断当前token的类型作出相应的处理，用到了两个函数：

    1. lookup()：功能是将一个多层嵌套的对象属性剥离开，返回其基础数据类型(例如，n.m.p=123，将’.’去除，最终返回123)；

    2. parseArray()：功能是将’#'和'/'之前的嵌套token展开并返回

  • lookup()函数的实现：将当前字符串中的’.’去除并保存新字符串中，依次向下获取，直到返回基础数据类型

  • parseArray()函数的实现：这函数要递归调用renderTemplate()函数，调用的次数由data决定
